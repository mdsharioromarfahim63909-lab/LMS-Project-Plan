# Version-Control Evidence — Screenshots

This folder holds the screenshots that prove real version-control usage. Capture each
image from GitHub, save it here using the exact file name below, and reference it in the
report as the matching figure number (see Section 7.7).

| File name | Figure | What it shows | How to capture it |
|-----------|--------|---------------|-------------------|
| `01-repo-creation.png` | 7.1 | The repository main page | Open the repo on GitHub — the landing page |
| `02-folder-structure.png` | 7.2 | The file/folder structure | Repo main page showing the `docs/` and `screenshots/` folders |
| `03-commit-history.png` | 7.3 | Multiple commits over time | Repo → **Commits** tab (click the "N commits" link) |
| `04-commit-detail.png` | 7.4 | A single commit's file changes | Click any commit to open its diff (green = added, red = removed) |
| `05-branches.png` | 7.5 | Branch usage | Repo → branch dropdown, or the **Branches** tab |
| `06-readme-rendered.png` | 7.6 | The README rendered by GitHub | Repo main page (GitHub renders `README.md` automatically) |
| `07-file-upload-evidence.png` | 7.7 | Member files inside the repo | Navigate into the `docs/` folder on GitHub |
| `08-contributors.png` | 7.8 | Contributor activity | Repo → **Insights** → **Contributors** |

## Notes

- Use PNG so the text stays sharp. Crop to the relevant area; keep the browser address bar
  visible where it helps prove the repository URL.
- The `05-branches.png` shot is easiest to capture right after the `feature/cm-documentation`
  branch is pushed — the branch will appear in the dropdown and the Branches tab.
- For `03-commit-history.png`, the history already contains a branch-and-merge graph, which
  is good evidence of the branching workflow described in Section 7.2.
- **Tip (optional):** adding teammates as collaborators (Settings → Collaborators) and
  having them make even one commit each strengthens the contributor evidence in
  `08-contributors.png`.
